# Sales & Stock Bot 📦📊 (Telegram + FastAPI V2)

Мощная и гибкая система в виде Telegram-бота для автоматизации учета товарных остатков, фиксации продаж сотрудниками и оперативного контроля для администратора. В версии V2 проект перерос рамки простого MVP и теперь включает **полноценную Web-аналитику**, **ролевую модель с премодерацией**, **управление категориями** и генерацию **Excel-отчетов**.

Проект построен на передовом асинхронном стеке с использованием **Aiogram 3.x**, **SQLAlchemy 2.0** и **FastAPI**.

---

## 🔥 Что нового в V2? (Основные фичи)

В текущей версии реализовано 5 крупных функциональных блоков:

1. **🔐 Премодерация сотрудников (White List)**
   - Любой новый пользователь, нажавший `/start`, получает статус `PENDING`.
   - Администратор получает мгновенное уведомление о новой заявке с инлайн-кнопками `[✅ Одобрить]` и `[⛔ Отклонить]`.
   - Только после одобрения (статус `WORKER`) сотрудник получает доступ к оформлению продаж.

2. **🗂 Категории товаров**
   - Товары на складе теперь группируются по категориям (создаются администратором в специальном меню "🗂 Категории").
   - Удобная навигация: при продаже сотрудник сначала выбирает нужную категорию, и только потом подходящий товар, что ускоряет работу с большой номенклатурой.

3. **⏪ Безопасный откат продаж (Rollback)**
   - Администратор получает уведомление о каждой продаже с кнопкой `[❌ Отменить (Rollback)]` прямо под сообщением.
   - Одно нажатие автоматически удаляет транзакцию из финансовой статистики и **строго возвращает проданное количество товара обратно на склад**.

4. **📥 Экспорт отчетов в Excel**
   - Генерация полноценных XLSX-файлов по клику кнопки в меню статистики.
   - Файлы формируются «на лету» в оперативной памяти с помощью `openpyxl` (без захламления дискового пространства сервера) и отправляются администратору прямо в чат.

5. **🌐 Интерактивный Web-Dashboard (FastAPI)**
   - Кнопка «🌐 Web Dashboard» открывает полноценное веб-приложение (Telegram WebApp) прямо внутри мессенджера (или в браузере).
   - Дашборд реализован с премиальным дизайном (Glassmorphism, Plus Jakarta Sans).
   - **Real-time обновления**: графики (Chart.js), выручка и рейтинг сотрудников (Топ-5) обновляются автоматически каждые 10 секунд без перезагрузки интерфейса.
   - Интерфейс локализован на узбекский язык 🇺🇿, цены приведены в сумах.

## 🎭 Ролевая модель и функционал

### 👑 Администратор (Владелец)
- **Управление складом**: Добавление категорий, товаров (FSM) и быстрое изменение остатков через inline-кнопки.
- **Статистика продаж**: Просмотр финансовых отчетов в боте (чеки, выручка, Топ-3) + выгрузка в **Excel**.
- **Система алертов**: Мгновенные уведомления о продажах и **Critical Stock Alert** ⚠️ (если остаток товара падает ниже 5 единиц).
- **Управление доступом**: Одобрение или блокировка новых сотрудников в 1 клик.
- **Web Analytics**: Доступ к живому дашборду через кнопку в меню.

### 👷 Сотрудник (WORKER)
- **Оформление продаж**: Пошаговый интерфейс: Категория -> Товар -> Количество (с защитой от отрицательных остатков).
- **Трекинг смены**: Личная статистика продаж за текущий день (чеки, проданные единицы, сумма).

## 🛠 Технологический стек

- **Язык:** Python 3.12+
- **Telegram Фреймворк:** [Aiogram 3.x](https://docs.aiogram.dev/)
- **Web Фреймворк:** [FastAPI](https://fastapi.tiangolo.com/) (REST API + отдача UI)
- **База Данных:** SQLite (асинхронный драйвер `aiosqlite`)
- **ORM:** [SQLAlchemy 2.0](https://docs.sqlalchemy.org/) (Strict Asyncio)
- **Миграции БД:** Alembic (включен `render_as_batch` для модификации схем в SQLite)
- **Конфигурация:** Pydantic Settings (чтение `app/config.py` из `.env`)
- **Excel генерация:** `openpyxl`
- **Frontend:** Vanilla JS, HTML, CSS (Chart.js для графиков)

## 📂 Структура проекта

Проект базируется на слоистой архитектуре с самописным независимым DI-контейнером:

```text
sales_and_stock_bot/
├── app/
│   ├── api/            # FastAPI приложение
│   │   ├── routers/    # REST API эндпоинты (stats.py)
│   │   ├── templates/  # Frontend UI (index.html)
│   │   └── server.py   # Инициализация FastAPI
│   ├── database/       # SQLAlchemy ядро и Модели (User, Product, Category, Transaction)
│   ├── services/       # Бизнес-логика (CRUD сервисы, Rollback логика)
│   ├── telegram/       
│   │   ├── keyboards/  # Reply и Inline клавиатуры
│   │   ├── routers/    # Хендлеры (admin.py, worker.py)
│   │   ├── states/     # FSM Состояния
│   │   ├── middlewares/# AuthMiddleware (Премодерация, внедрение DI)
│   │   └── bot.py      # Инициализация Aiogram
│   ├── app.py          # Точка сборки, запуск асинхронных тасок (Telegram + API)
│   ├── config.py       # Pydantic настройки проекта
│   └── container.py    # DI-контейнер
├── migrations/         # Миграции Alembic
├── alembic.ini         # Конфигурация Alembic
├── main.py             # Точка входа
├── .env                # Настройки окружения
└── requirements.txt    # Зависимости
```

## 🚀 Установка и Запуск

1. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/vash_username/sales_and_stock_bot.git
   cd sales_and_stock_bot
   ```

2. **Установите зависимости**:
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # На Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **Настройте переменные окружения**:
   Создайте файл `.env` в корне проекта на основе ваших данных:
   ```env
   # Токен Telegram-бота (от @BotFather)
   BOT_TOKEN=1234567890:YOUR_TELEGRAM_BOT_TOKEN
   
   # Telegram ID Администратора (получить у @userinfo_bot)
   ADMIN_ID=123456789
   
   # Включение модулей
   RUN_TELEGRAM=True
   RUN_API=True
   
   # Настройки FastAPI
   API_HOST=0.0.0.0
   API_PORT=8000
   
   # URL для Telegram WebApp (по умолчанию локалхост с https, 
   # для реальной работы через ngrok укажите ваш домен)
   WEBAPP_URL=https://localhost:8000
   ```

4. **Примените миграции (Инициализация БД)**:
   ```bash
   python -m alembic upgrade head
   ```

5. **Запустите проект**:
   ```bash
   python main.py
   ```
   *Вы увидите в логах успешный старт как Telegram-бота, так и сервера Uvicorn (FastAPI)*.

## 🖥 Использование WebApp в Telegram

Из-за строгой политики безопасности Telegram, Web Apps работают только по HTTPS протоколу. 
Для тестирования на локальной машине с телефона:
1. Запустите [ngrok](https://ngrok.com/): `ngrok http 8000`
2. Скопируйте полученную `https://...` ссылку.
3. Вставьте её в `.env` файл в параметр `WEBAPP_URL`.
4. Перезапустите бота. Кнопка **🌐 Web Dashboard** в панели администратора начнет открывать живой интерфейс внутри Telegram!
