# Sales & Stock Bot 📦📊

Telegram-бот для автоматизации учета товарных остатков, фиксации продаж сотрудниками и оперативного контроля для администратора (MVP).
Проект построен на асинхронном стеке с использованием Aiogram 3.x и SQLAlchemy 2.0.

## Особенности и Функционал

Проект разделен на две основные роли на основе Telegram ID пользователя:

### 👑 Администратор (Владелец)
- **Управление складом**: Добавление новых товаров через пошаговый мастер (FSM) и быстрое изменение остатков (пополнение/списание) через inline-кнопки.
- **Статистика продаж**: Просмотр финансовых отчетов за сегодня и за неделю (чеки, единицы товаров, выручка, Топ-3 самых продаваемых позиций).
- **Система алертов**: Получение мгновенных уведомлений о каждой новой продаже.
- **Critical Stock Alert**: Отдельные предупреждения `⚠️` в случае, если после продажи остаток товара на складе становится меньше 5 единиц.
- **Сотрудники**: Просмотр списка всех зарегистрированных сотрудников в системе.

### 👷 Сотрудник 
- **Оформление продаж**: Быстрый и удобный интерфейс с inline-кнопками для выбора товара и ввода количества (весь процесс занимает до 10 секунд).
- **Жесткая валидация**: Система не позволит продать больше единиц товара, чем есть на складе в данный момент (защита от отрицательного остатка).
- **Трекинг смены**: Возможность посмотреть свои личные продажи и сумму выручки за текущий день. Запись всех действий закрепляется за конкретным сотрудником.

## 🛠 Технологический стек

- **Язык:** Python 3.12+
- **Фреймворк:** [Aiogram 3.x](https://docs.aiogram.dev/) (Telegram Bot API)
- **База Данных:** SQLite (через драйвер `aiosqlite`)
- **ORM:** [SQLAlchemy 2.0](https://docs.sqlalchemy.org/) (Asyncio)
- **Миграции:** Alembic
- **Конфигурация:** Pydantic Settings (`.env`)
- **DI (Внедрение зависимостей):** Самописный контейнер (`app/container.py`), пробрасываемый в хендлеры через middleware.

## 📂 Архитектура и Структура

Проект следует принципам чистой архитектуры и слоям:

```text
sales_and_stock_bot/
├── app/
│   ├── database/       # Подключение к БД, базовые классы и SQLAlchemy модели (User, Product, Transaction)
│   ├── services/       # Бизнес-логика (ProductService, TransactionService, UserService)
│   ├── telegram/       
│   │   ├── keyboards/  # Reply и Inline клавиатуры
│   │   ├── routers/    # Aiogram Routers (Отдельные роутеры для Admin и Worker)
│   │   ├── states/     # FSM Состояния
│   │   ├── middlewares/# Aiogram Middlewares (AuthMiddleware - проверка ролей)
│   │   └── bot.py      # Инициализация бота и Dispatcher
│   ├── app.py          # Точка сборки (Инициализация DI-контейнера и запуск)
│   ├── config.py       # Pydantic настройки проекта
│   └── container.py    # Простой DI-контейнер
├── migrations/         # Директория миграций Alembic
├── alembic.ini         # Конфигурация Alembic
├── main.py             # Точка входа в приложение
├── .env                # Настройки окружения (токены, ID админа)
└── requirements.txt    # Зависимости Python
```

## 🚀 Установка и Запуск (Локально)

1. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/vash_username/sales_and_stock_bot.git
   cd sales_and_stock_bot
   ```

2. **Установите зависимости**:
   Рекомендуется использовать виртуальное окружение (`venv`).
   ```bash
   pip install -r requirements.txt
   ```

3. **Настройте переменные окружения**:
   Отредактируйте или создайте файл `.env` в корне проекта со следующим содержимым:
   ```env
   # Токен Telegram-бота (получить у @BotFather)
   BOT_TOKEN=1234567890:YOUR_TELEGRAM_BOT_TOKEN
   
   # Telegram ID Администратора (получить у @userinfobot)
   # Только пользователь с этим ID получит права владельца. Остальные станут WORKER.
   ADMIN_ID=123456789
   ```

4. **Примените миграции (Инициализация БД)**:
   При первой установке вам нужно создать таблицы базы данных:
   ```bash
   python -m alembic upgrade head
   ```
   *Это создаст файл `app.db` со всей схемой (sqlite3).*

5. **Запустите бота**:
   ```bash
   python main.py
   ```
   *Вы должны увидеть в консоли `INFO - Starting Telegram Bot...`*

## 🤖 Использование (Тестирование)

1. Отправьте боту команду `/start`.
2. Если ваш `Telegram ID` совпадает с указанным в `.env` (значение `ADMIN_ID`), вы получите приветствие **"Добро пожаловать в панель администратора"** и клавиатуру владельца (📦 Склад, 👥 Сотрудники, 📊 Статистика).
   - *Попробуйте добавить товар в Склад-меню через кнопку "➕ Добавить товар".*
3. Если зайти с другого аккаунта Telegram (чей ID не указан как `ADMIN_ID`), пользователь автоматически получит статус **WORKER**.
   - Ему будет доступно меню сотрудника (🛒 Оформить продажу, 📈 Мои продажи).
4. Оформите тестовую продажу со второго аккаунта и проверьте, как в аккаунт Администратора приходит ⚠️ **Алерт** о покупке.
