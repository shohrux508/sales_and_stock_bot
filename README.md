# Sales & Stock Bot 📦📊 (Telegram + FastAPI V3)

Мощная и гибкая система в виде Telegram-бота для автоматизации учета товарных остатков, фиксации продаж сотрудниками и оперативного контроля для администратора. В версии V3 проработана **глубокая бизнес-логика**: полноценное **управление запасами** (приемка/списание), **мульти-корзина покупок** и интеграция со **штрих-кодами**.

Проект построен на передовом асинхронном стеке с использованием **Aiogram 3.x**, **SQLAlchemy 2.0**, **FastAPI** и покрыт тестами на **pytest**.

---

## 🖨️ Что нового в V4? (Автоматическая печать чеков)

Версия V4 добавляет мгновенную печать товарных чеков на термопринтере Xprinter при каждой продаже:

1. **🖨️ Автоматическая печать чеков**
   - При оформлении продажи сотрудником чек мгновенно отправляется на термопринтер Xprinter через WebSocket.
   - Архитектура **Cloud → Local**: сервер в облаке транслирует данные на локальный ПК с USB-принтером.
   - Поддержка **кириллицы и узбекской латиницы** в чеке.

2. **📡 WebSocket-канал связи**
   - Защищённый эндпоинт `/ws/printer/{secret_token}` для подключения клиента-принтера.
   - Автоматическое переподключение при обрыве связи (каждые 5 секунд).
   - Redis-дедупликация — один order_id не может быть напечатан дважды.

3. **🔄 Fallback: кнопка ручной печати**
   - Если принтер не подключен в момент продажи — чек сохраняется в очередь.
   - Администратор получает кнопку **"🖨️ Chekni chop etish"** в уведомлении о продаже.
   - При подключении принтера — нажатие кнопки отправляет чек на печать.

## 🚀 Что нового в V3? (Бизнес-логика)

Версия V3 сфокусирована на реальных складских и торговых процессах:

1. **📦 Управление запасами (Приемка и Списание)**
   - Выделены отдельные бизнес-процессы для Поступления товара (+) и Списания/Брака (-) с обязательным указанием причины списания.
   - Финансовая статистика (Выручка) строго сепарирована от складских движений.
   - Меню администратора дополнено удобными кнопками "📥 Приемка" и "🗑 Списание".

2. **🛒 Корзина продаж (Мульти-чеки)**
   - Сотрудники теперь могут добавлять несколько разных товаров в один заказ (чек) в цикле.
   - Транзакции в БД связываются единым `order_group_id`, что позволяет анализировать комплексные продажи.
   - При продаже администратору отправляется сводная квитанция (чек) с итоговой суммой по всему списку товаров.
   - Отмена транзакции откатывает товар верно в зависимости от его типа.

3. **🏷 Интеграция штрих-кодов**
   - В карточке товара (при редактировании) появилась возможность привязать уникальный штрихкод.
   - **Автоматический поиск:** Сотрудник может отсканировать штрихкод физическим сканером прямо в чат бота (или отправить текстом от 8 до 14 цифр), и бот мгновенно переведет его на этап оформления продажи данного товара, минуя каталоги и категории.

## 🔥 Что было добавлено в V2? (Архитектурные фичи)

В версии V2 было реализовано 5 крупных функциональных блоков:

1. **🔐 Премодерация сотрудников (White List)**
   - Любой новый пользователь, нажавший `/start`, получает статус `PENDING`.
   - Администратор получает мгновенное уведомление о новой заявке с инлайн-кнопками `[✅ Одобрить]` и `[⛔ Отклонить]`.
   - Только после одобрения (статус `WORKER`) сотрудник получает доступ к оформлению продаж.

2. **🗂 Категории товаров**
   - Товары на складе теперь группируются по категориям (создаются администратором в специальном меню "🗂 Категории").
   - Удобная навигация: при продаже сотрудник сначала выбирает нужную категорию, и только потом подходящий товар, что ускоряет работу с большой номенклатурой.

3. **⏪ Безопасный откат продаж (Rollback)**
   - Администратор получает уведомление о каждой продаже с кнопкой `[❌ Отменить (Rollback)]` прямо под сообщением.
   - Одно нажатие автоматически удаляет транзакцию из финансовой статистики и **строго возвращает проданное количество товара обратно на склад**.

4. **📥 Экспорт отчетов в Excel**
   - Генерация полноценных XLSX-файлов по клику кнопки в меню статистики.
   - Файлы формируются «на лету» в оперативной памяти с помощью `openpyxl` (без захламления дискового пространства сервера) и отправляются администратору прямо в чат.

5. **🌐 Интерактивный Web-Dashboard (FastAPI)**
   - Кнопка «🌐 Web Dashboard» открывает полноценное веб-приложение (Telegram WebApp) прямо внутри мессенджера (или в браузере).
   - Дашборд реализован с премиальным дизайном (Glassmorphism, Plus Jakarta Sans).
   - **Real-time обновления**: графики (Chart.js), выручка и рейтинг сотрудников (Топ-5) обновляются автоматически каждые 10 секунд без перезагрузки интерфейса.
   - Интерфейс локализован на узбекский язык 🇺🇿, цены приведены в сумах.

## 🎭 Ролевая модель и функционал

### 👑 Администратор (Владелец)
- **Управление складом**: Добавление категорий, товаров (FSM) и быстрое изменение остатков через inline-кнопки.
- **Статистика продаж**: Просмотр финансовых отчетов в боте (чеки, выручка, Топ-3) + выгрузка в **Excel**.
- **Система алертов**: Мгновенные уведомления о продажах и **Critical Stock Alert** ⚠️ (если остаток товара падает ниже 5 единиц).
- **Управление доступом**: Одобрение или блокировка новых сотрудников в 1 клик.
- **Web Analytics**: Доступ к живому дашборду через кнопку в меню.

### 👷 Сотрудник (WORKER)
- **Оформление продаж**: Пошаговый интерфейс: Категория -> Товар -> Количество (с защитой от отрицательных остатков).
- **Трекинг смены**: Личная статистика продаж за текущий день (чеки, проданные единицы, сумма).

## 🛠 Технологический стек

- **Язык:** Python 3.12+
- **Telegram Фреймворк:** [Aiogram 3.x](https://docs.aiogram.dev/)
- **Web Фреймворк:** [FastAPI](https://fastapi.tiangolo.com/) (REST API + отдача UI)
- **База Данных:** SQLite (асинхронный драйвер `aiosqlite`)
- **ORM:** [SQLAlchemy 2.0](https://docs.sqlalchemy.org/) (Strict Asyncio)
- **Миграции БД:** Alembic (включен `render_as_batch` для модификации схем в SQLite)
- **Конфигурация:** Pydantic Settings (чтение `app/config.py` из `.env`)
- **Excel генерация:** `openpyxl`
- **Тестирование:** `pytest`, `pytest-asyncio`
- **Frontend:** Vanilla JS, HTML, CSS (Chart.js для графиков)

## 🚀 Установка и Запуск

1. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/vash_username/sales_and_stock_bot.git
   cd sales_and_stock_bot
   ```

2. **Установите зависимости**:
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # На Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **Настройте переменные окружения**:
   Создайте файл `.env` в корне проекта на основе ваших данных:
   ```env
   # Токен Telegram-бота (от @BotFather)
   BOT_TOKEN=1234567890:YOUR_TELEGRAM_BOT_TOKEN
   
   # Telegram ID Администраторов (через запятую, получить у @userinfo_bot)
   ADMIN_IDS=123456789,987654321
   
   # Включение модулей
   RUN_TELEGRAM=True
   RUN_API=True
   
   # Настройки FastAPI
   API_HOST=0.0.0.0
   API_PORT=8000
   
   # URL для Telegram WebApp (по умолчанию локалхост с https, 
   # для реальной работы через ngrok укажите ваш домен)
   WEBAPP_URL=https://localhost:8000
   ```

4. **Примените миграции (Инициализация БД)**:
   ```bash
   python -m alembic upgrade head
   ```

5. **Запустите проект**:
   ```bash
   python main.py
   ```
   *Вы увидите в логах успешный старт как Telegram-бота, так и сервера Uvicorn (FastAPI)*.

## 🐳 Деплой через Docker (Рекомендуется)

Самый быстрый и надежный способ запустить проект — использовать Docker.

1. **Сборка и запуск**:
   ```bash
   docker-compose up --build -d
   ```
   Этот процесс автоматически применит миграции базы данных и запустит приложение.

2. **Просмотр логов**:
   ```bash
   docker-compose logs -f
   ```

### 🚄 Деплой на Railway / Render / DigitalOcean
Проект полностью готов к деплою на облачные платформы:
- При деплое укажите `DATABASE_URL` в переменных окружения (например, для Railway это может быть `sqlite+aiosqlite:///app.db`).
- `Dockerfile` сам выполнит миграции при старте через `entrypoint.sh`.

## 🖥 Использование WebApp в Telegram

Из-за строгой политики безопасности Telegram, Web Apps работают только по HTTPS протоколу. 
Для тестирования на локальной машине с телефона:
1. Запустите [ngrok](https://ngrok.com/): `ngrok http 8000`
2. Скопируйте полученную `https://...` ссылку.
3. Вставьте её в `.env` файл в параметр `WEBAPP_URL`.
4. Перезапустите бота. Кнопка **🌐 Web Dashboard** в панели администратора начнет открывать живой интерфейс внутри Telegram!

## 🖨️ Настройка печати чеков (Xprinter)

Модуль печати работает по схеме **Cloud Server → Local Client**. Сервер отправляет данные о продаже через WebSocket, а локальный клиент принимает их и отправляет на USB-принтер.

### Подготовка принтера

1. **Подключите Xprinter** к ПК через USB.
2. **Установите WinUSB драйвер** через [Zadig](https://zadig.akeo.ie/) (для режима USB), или используйте **стандартный Generic/Text** драйвер Windows.
3. **Узнайте Vendor/Product ID** принтера через Диспетчер устройств → Свойства → ИД оборудования.

### Запуск клиента

```bash
# 1. Скопируйте папку printer_client/ на ПК с принтером
# 2. Отредактируйте printer_client/config.py:
#    SECRET_TOKEN = "ваш-токен-из-.env"  (PRINTER_SECRET_TOKEN)
#    PRINTER_VENDOR_ID = 0x0416          (по модели принтера)
#    PRINTER_PRODUCT_ID = 0x5011         (по модели принтера)

# 3. Установите зависимости
cd printer_client
pip install -r requirements.txt

# 4. Запустите клиент
python client.py
# Или дважды кликните start_printer.bat для автозапуска с перезапуском
```

> **Тестовый режим**: Клиент работает и без физического принтера — чеки выводятся в консоль.
> При подключении USB-принтера автоматически переключится на реальную печать.

### Поведение при отсутствии принтера

Если принтер не подключен в момент продажи:
- Чек сохраняется в очередь на сервере.
- Администратор получает уведомление с кнопкой **"🖨️ Chekni chop etish"**.
- Нажатие кнопки после подключения принтера → чек отправляется на печать.
