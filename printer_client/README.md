# 🖨️ Sale & Stock Bot — Printer Client

Клиентское приложение для автоматической печати товарных чеков на термопринтере (Xprinter) через WebSocket.

## 🚀 Основные возможности
- **Автономность**: Автоматическое подключение к серверу и переподключение при обрыве связи.
- **Поддержка Xprinter**: Работает в двух режимах:
  - **USB**: Прямая работа через `python-escpos` и `pyusb` (требует драйвера WinUSB).
  - **Windows**: Печать через стандартный драйвер принтера Windows.
- **Удаленное управление**: Печать инициируется сервером (через Telegram-бот) в режиме реального времени.
- **Обработка ошибок**: Отправляет статус выполнения обратно на сервер (ACK/ERROR).

## 🛠️ Установка и настройка

### 1. Требования
- **Python 3.12+**
- Термопринтер (например, Xprinter XP-58IIH или XP-365B)

### 2. Установка зависимостей
Запустите `start_printer.bat`, он автоматически установит всё необходимое. 
Или вручную:
```bash
pip install -r requirements.txt
```

### 3. Настройка (config.py)
Откройте файл `config.py` и проверьте следующие параметры:
- `SERVER_WS_URL`: URL вашего сервера (например, `wss://your-app.up.railway.app/ws/printer/{token}`).
- `SECRET_TOKEN`: Секретный токен (должен совпадать с `PRINTER_SECRET_TOKEN` на сервере).
- `PRINTER_MODE`: 
    - `"usb"` — рекомендуется для прямой работы.
    - `"windows"` — если используете стандартный драйвер печати.
- `PRINTER_VENDOR_ID` и `PRINTER_PRODUCT_ID`: Если используете USB-режим, укажите ID вашего принтера (через Диспетчер устройств или Zadig).

## 🖨️ Настройка USB-принтера (Zadig)
Если вы используете режим `usb`, необходимо заменить стандартный драйвер принтера на **WinUSB**:
1. Скачайте [Zadig](https://zadig.akeo.ie/).
2. В меню выберите `Options > List All Devices`.
3. Выберите ваш принтер в списке (например, `XP-58`).
4. Выберите драйвер `WinUSB` и нажмите `Replace Driver`.

## ▶️ Запуск
Просто дважды кликните по файлу **`start_printer.bat`**.

Клиент запустится в консольном окне, подключится к серверу и будет ждать заданий на печать. Если принтер не подключен, чек будет выведен в консоль для отладки.

## 📝 Структура файлов
- `client.py` — Основная логика WebSocket и обработки сообщений.
- `receipt_printer.py` — Формирование макета чека и управление принтером.
- `config.py` — Настройки.
- `start_printer.bat` — Скрипт автоматического запуска.
